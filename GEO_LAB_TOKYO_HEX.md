# 2026-05-31 Tokyo Hex Geo Lab

## 目标

在博客中增加一个和 GIS / QGIS / Web 可视化相关，但不依赖访客当前位置的长期内容页面。

页面路径：`/geo-lab/tokyo-hex`

## 当前实现

- 使用 QGIS / PyQGIS 读取公开 GIS 数据，生成东京 23 区范围内的 hex grid。
- 输出 Web 前端可直接读取的 `public/data/tokyo-hex-cells.geojson`。
- 前端用 SVG 展示文明风格六边形地图，支持拖拽、缩放、筛选和点击查看详情。
- 右下角 Live2D 小人可以通过 `assistant:say` 事件对选中的格子做规则型吐槽。

## 数据来源

- 国土数值信息：东京行政界，用于抽出东京 23 区范围。
- 东京都公开数据：绿地/公园、水系图层，用于判定市立公园和河川。
- 文明百科本地页面资源：用于原型阶段的区域/改良图标。

## 分类边界

当前真实接入的 GIS 判定：

- `river`：水系图层相交。
- `park`：绿地/公园图层相交。
- `urban_plain`：没有更强 GIS 信号时的市街地 fallback。

当前只作为模型预留、不会用假规则生成的分类：

- 学院
- 工业区
- 商业中心
- 剧院广场
- 圣地
- 港口
- 社区
- 农场
- 铁路

这些分类后续应通过 OSM POI / landuse / railway 等公开数据计数来生成。

## 隐私设计

- 不请求浏览器 geolocation。
- 不使用访客 IP 定位。
- 不保存访客位置。
- 页面只展示公开 GIS 数据经过 QGIS 处理后的抽象结果。
